- **Data-quality guardrails (PRD M2, added 2026-07-09; all read-only, `file:...?mode=ro`):** `check_coverage.py` (fails if latest-day close count < `max(5000, 90%*10-day-median)`; wired into `daily.bat` after refresh, before MTM — `exit /b 1` skips all MTM+overlay ops on a bad-data day), `check_anomalies.py` (flags held >300% / cache-wide >1000%-above-$1 single-day moves + missing held marks → `var/anomaly_report.log`; wired after MTM, non-blocking always-exit-0), `check_cache_gaps.py` (rankable tickers with internal history holes >5 trading days over a 13-mo window → `var/cache_gap_report.log`; standalone monthly). All three pick "real trading days" via a `MIN_TRADING_DAY_COUNT=1000` filter to exclude market-closed noise days. (record BB–BE, verified this session)
- **Backup hygiene (PRD M5, added 2026-07-09):** `scripts/backup_trades_db.py` — SQLite `VACUUM INTO` (never a bare copy of the live WAL DB) → `var/backups/trades_YYYY-MM-DD.db`, keeps newest `--keep` (default 3), aborts if free disk < 2x DB size, never touches `var/trades.db.bak_pre_spike_cleanup`. Weekly `TradingWeeklyBackup` scheduled task (Sun 9am → `var/backup.log`). Restore = plain file copy of a backup + open; `verify_run --db <copy>` confirms fidelity. Backup ~4.76 GB, ~60s. (record BM)
- **Experiment-integrity reporting (PRD M4, added 2026-07-09):** `experiment_report.py` (read-only) — per LLM-overlay experiment (stock=`llm_overlay_log`, sector=`sector_overlay_log`; each shared by control/cash-veto/cascade sleeves): kill-switch counters (decisions vs 30, months vs 12 from FIRST decision not the 07-01 reset), unrealized score-vs-forward-return table, and control-vs-treatment NAV divergence (%-from-inception + gap $/pp, all 6 sleeves share 07-06 inception/$100k). - **Unattended-automation safety (PRD M3, added 2026-07-09):** `paper_mtm.py` now has a **pre-inception NAV guard** — `inception_date(strategy) = min(date(initialized_at), earliest entry_date)` (neither source alone is right: initialized_at is too late for backdated sleeves, entry_date too late for the 07-06 cohort's next-open fills), defensive `date.min` fallback; skips writing any `paper_nav` row before inception. `verify_run.py --mode daily|monthly` (read-only): per-sleeve NAV continuity / cash recon / position-count (monthly) / no-pre-inception → `var/verify_report.log`, `--db` flag, report co-located with the DB; wired into `daily.bat` (after MTM, FAIL→exit/b1) and `monthly_auto.bat`. `ops_stamp.py` writes a dated one-liner to `var/ops_status.log` (NOT `daily_report.md`, which is Evan's hand-written journal). Regression test `test_inception_guard.py`. (record BF–BI, verified this session)
- **Self-healing daily MTM (PRD amendment M3.5, 2026-07-09):** `mtm_catchup.py` marks every SETTLED missing trading day (today included) for all sleeves; guards = missing + coverage-settled + post-inception + post-last-rebalance; **never overwrites an existing NAV** (only fills gaps). `check_coverage.coverage_status()` is the shared floor logic (gate + catch-up + verify). `verify_run` continuity checks only to the last SETTLED day (below-floor today = PENDING, not a gap). `daily.bat` rewritten (goto flow): refresh → [stops if today settled] → mtm_catchup → anomaly → graphify → verify → stamp; **exits 0 on a normal pending day** (today heals next run), fails only on a real gap/error. Exit codes: mtm_catchup 0=today marked, 2=today pending, 1=error. (record BN)
- **`paper_mtm` is coverage-gated too (2026-07-10, record BQ):** `paper_mtm.main()` refuses to mark an `--as-of` day whose coverage is below the floor (exit 2, no write) unless `--force` — closes the raw-MTM bypass (a concurrent session had marked 07-09 sub-floor via raw `paper_mtm`, record BP). The floor logic lives once in `check_coverage.coverage_status()`, consumed by the daily gate, `mtm_catchup`, `verify_run` (pending boundary), and `paper_mtm`. **Only one owner should mark NAVs (the daily.bat catch-up)** — a process convention, not code-enforced; other sessions/humans running raw `paper_mtm` against live is discouraged.
