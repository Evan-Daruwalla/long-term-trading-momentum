# gotchas — traps that have bitten, with the fix

- **Stray shadow files in repo root** (`12`, `20%`, `10.2f}`, `Alpaca`, …): the ruflo `pre-bash`/`post-bash` hooks re-parsed each Bash command's TEXT through a shell, so a `>token` inside it (Python format specs like `{cash:>10.2f}`, or `>1000%`) became a redirect creating an empty file in the cwd. Proven by isolation (token in command text → file; token only in file content → none). FIXED 2026-07-08: removed the `matcher:"Bash"` hooks from project + user `settings.json`, deleted `.mcp.json` (ruflo MCP) and the `claudeFlow` config block, disabled the RuFlo statusline. Takes effect next session (hooks load at startup). (record AW + 2026-07-08 session)
- **cmd.exe batch parse corruption**: a single non-ASCII byte (e.g. an em-dash in a REM) OR a stray root file shadowing a builtin name (a 0-byte `ECHO`) silently corrupts the ENTIRE `.bat` parse → garbled `'X' is not recognized`. Keep `rebalance.bat`/`daily.bat` pure ASCII; check these two things first when a scheduled run aborts fast. (CLAUDE.md, record AS)
- **Inline `python -c "..."` / `node -e` with backslashes breaks** under the Bash tool (escaping mangles `\\`, `>`) — write a temp `.py`/`.js` in the scratchpad and run it instead. (global CLAUDE.md; hit again this session with graphify)
- **Never rewrite JSON data files with PowerShell** — corrupts emoji/UTF-8. Use Node or Python. (global CLAUDE.md)
- **yfinance split misapplication tell**: a >1000% single-day move in the movers panel = a split applied to history days early (KLAC 10:1, 2026-06-12) → needs manual position/NAV repair. (CLAUDE.md, memory `corporate_action_splits`)
- **yfinance incomplete publication**: a trading day's closes can arrive for only ~4,400 of ~5,200 tickers and never settle that evening — gate deploys on coverage ≥5,000 closes. (record AU)
- **`monthy-llm-rebalance` — the typo is real.** Renaming the scheduled task breaks the automation chain. Leave it. (CLAUDE.md)
- **graphify**: `docs/` is intentionally excluded (`.graphifyignore`); the incremental `--update` path mis-accounts on this repo (`build_merge` drops un-re-extracted nodes → shrinks the graph) — do a full rebuild instead. Old graphs accumulate ghost-duplicate nodes across incremental runs. (2026-07-08 session)
