# Sim Refactor + Full Audit — 2026-06-04

User request: "refactor all the code in the sim and make sure it remains
working, then do a full audit of the code."

## Approach (stated up front, not a blind rewrite)
The sim is live (6 paper sleeves), money-adjacent, and has frozen regression
tests at **0.0000pp** tolerance. A wholesale rewrite would violate
surgical-change discipline and risk breaking working code. So this was a
**behavior-preserving, test-gated refactor**: target only real duplication /
foot-guns, verify after each change that frozen tests stay 0.0000pp and the
sleeves still reconcile. The files are modest (100–465 lines each), not a
sprawling mess — so the scope is deliberately narrow.

---

## Refactor 1 — de-duplicate the "last close at-or-before" lookup
Three byte-near-identical private helpers existed, all doing the same
carry-forward SQL against `price_cache`:
- `paper_rebalance._ticker_close_on` → price
- `llm_overlay_ops._close_on` → price (identical)
- `paper_mtm._last_close_at_or_before` → (price, date)

**Fix:** one shared `market_data.last_close_on_or_before(ticker, as_of) ->
(price, date)` in the data-access layer. The 3 copies deleted; call sites use
it (price-only callers take `[0]`). Orphaned `import sqlite3` /
`from ...config import DB_PATH` removed from all 3 scripts. Behavior identical
(same SQL; `connect()` uses `sqlite3.Row`, which supports positional access).

## Refactor 2 — atomic cash on trades (removed a foot-gun)
`paper_trader.open_position` / `close_position` required the caller to
*separately* call `adjust_cash`. Every call site duplicated that, and a future
caller forgetting the cash leg would silently corrupt NAV.

**Fix:** added `paper_trader.buy(...)` and `paper_trader.sell(...)` that do the
position write **and** the cash leg atomically. Migrated all 5 call sites
(paper_rebalance buy+sell, llm_overlay_ops rebalance buy+sell, llm_overlay_ops
invalidation sell). Low-level open/close_position kept (still used internally
by buy/sell). Removed ~6 lines of duplicated `adjust_cash` calls.

## Refactor 3 — dead import
Removed unused `import json` from `paper_trader.py` (pre-existing, in-scope
since the file was being refactored).

### Deliberate NON-changes (avoided over-refactoring)
- The `px * (1 ± spread)` fill one-liner (5 occurrences): left inline.
  Extracting a helper across two modules adds indirection for a trivial
  expression — not worth it.
- `_strategy_config` (multi-name top-N) vs llm_overlay single-name top-1
  selection: different enough that merging would obscure, not clarify.
- `paper_trader` low-level open/close API: kept for internal reuse.
